# Implicit Neural Representation for the Double Pendulum

This demo represents a double-pendulum trajectory with a neural network

$$
t \mapsto [\theta_1(t), \theta_2(t), \omega_1(t), \omega_2(t)].
$$

Instead of stepping the system forward with a numerical integrator at inference time, the model directly predicts the state at any queried time.

The folder contains three training variants:

- `supervised`: fit the INR to the full trajectory generated by the discrete solver.
- `physics`: fit the INR from the ODE residual plus the initial condition.
- `hybrid`: combine the physics loss with sparse samples from the discrete trajectory.

## Implementation

The main implementation lives in [src/ai_for_sim/inr.py](src/ai_for_sim/inr.py).

Current model and training behavior:

- The network is `TimeMLP`, a residual MLP with SiLU activations and 4 outputs.
- Time is normalized to `[-1, 1]` before entering the network.
- The input uses a fixed Fourier-style encoding with 16 sine features and 16 cosine features.
- Hidden activations are normalized inside the residual blocks and again before the output layer.
- The default demo uses `DoublePendulumParams(duration=6.0, fps=20)`.

Default quick configs used by the scripts:

- `supervised`: Adam, `epochs=1000`, `lr=2e-3`.
- `physics`: LBFGS, `epochs=500`, `lr=0.5`, `collocation_points=241`.
- `hybrid`: LBFGS, `epochs=500`, `lr=0.5`, `collocation_points=241`, `sparse_points=17`.

## Losses

### Supervised

The supervised variant minimizes MSE against the full reference trajectory from the numerical solver in [00-discrete-solver](00-discrete-solver).

### Physics

The physics-informed variant computes

- the time derivative of the network output with autograd,
- the double-pendulum right-hand side from the same predicted state,
- a residual loss between them,
- an initial-condition loss at `t = 0`.

The current implementation uses `torch.nn.functional.huber_loss` for both the residual term and the initial-condition term.

Let the INR prediction be

$$
\hat{\mathbf{y}}(t)
=
[\hat{\theta}_1(t), \hat{\theta}_2(t), \hat{\omega}_1(t), \hat{\omega}_2(t)],
$$

and let the double-pendulum ODE be written as

$$
\frac{d\mathbf{y}}{dt} = \mathbf{f}(\mathbf{y}).
$$

For collocation times $\{t_i\}_{i=1}^{N_c}$, the residual is

$$
\mathbf{r}(t_i)
=
\frac{d\hat{\mathbf{y}}}{dt}(t_i) - \mathbf{f}(\hat{\mathbf{y}}(t_i)).
$$

The physics-informed loss is

$$
\mathcal{L}_{\text{physics}}
=
\lambda_{\text{res}} \mathcal{L}_{\text{res}}
+
\lambda_{\text{ic}} \mathcal{L}_{\text{ic}},
$$

with

$$
\mathcal{L}_{\text{res}}
=
\frac{1}{N_c}\sum_{i=1}^{N_c}
\operatorname{Huber}(\mathbf{r}(t_i), \mathbf{0}),
\\
\mathcal{L}_{\text{ic}}
=
\operatorname{Huber}(\hat{\mathbf{y}}(0), \mathbf{y}_0).
$$

Unlike the earlier README draft, collocation points are currently sampled on a uniform grid with `torch.linspace(0, duration, collocation_points)`, not randomly.

### Hybrid

The hybrid variant adds a sparse data term:

$$
\mathcal{L}
=
\lambda_{\text{res}} \mathcal{L}_{\text{res}}
+
\lambda_{\text{ic}} \mathcal{L}_{\text{ic}}
+
\lambda_{\text{data}} \mathcal{L}_{\text{data}}.
$$

For sparse observation times $\{t_j^{(s)}\}_{j=1}^{N_s}$ with states $\{\mathbf{y}_j^{(s)}\}_{j=1}^{N_s}$,

$$
\mathcal{L}_{\text{data}}
=
\frac{1}{N_s}\sum_{j=1}^{N_s}
\operatorname{Huber}(\hat{\mathbf{y}}(t_j^{(s)}), \mathbf{y}_j^{(s)}).
$$

Here:

- `L_res` is the Huber loss between `d/dt` of the INR and the ODE right-hand side.
- `L_ic` is the Huber loss on the initial state.
- `L_data` is the Huber loss on sparse samples taken from the numerical trajectory.

Sparse supervision points are chosen by evenly spaced indices from the reference rollout.

## Scripts

Training scripts save checkpoints under [01-implicit-neural-representation/artifacts](01-implicit-neural-representation/artifacts):

- `train_supervised.py` -> `artifacts/supervised/supervised_checkpoint.pt`
- `train_physics.py` -> `artifacts/physics/physics_checkpoint.pt`
- `train_hybrid.py` -> `artifacts/hybrid/hybrid_checkpoint.pt`

Export scripts load a checkpoint and generate:

- `*_prediction.npz`
- `*_summary.png`
- `*_loss.png`
- `*_animation.gif`
- `*_comparison.gif`

The `.npz` file stores predicted states and, when available, the reference trajectory.

## Commands

Run the supervised INR:

```bash
uv run python 01-implicit-neural-representation/train_supervised.py
uv run python 01-implicit-neural-representation/export_supervised.py
```

Run the physics-informed INR:

```bash
uv run python 01-implicit-neural-representation/train_physics.py
uv run python 01-implicit-neural-representation/export_physics.py
```

Run the hybrid INR:

```bash
uv run python 01-implicit-neural-representation/train_hybrid.py
uv run python 01-implicit-neural-representation/export_hybrid.py
```

Run all three pipelines:

```bash
uv run python 01-implicit-neural-representation/simulate.py
```

Run the INR tests:

```bash
uv run pytest tests/test_inr_demo.py
```
